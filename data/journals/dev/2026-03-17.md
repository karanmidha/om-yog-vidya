- DOMPurify requires DOM, doesn't work in Node/test environment
- Created environment-aware sanitization (regex in Node, DOMPurify in browser)
- Alternative: Use isomorphic-dompurify for universal compatibility

**3. Rate Limiting Strategy:**
- Vercel Edge middleware is powerful for rate limiting
- In-memory storage works for basic scenarios but production should use Redis/KV
- Different endpoints need different rate limits (contact vs booking vs global)

**4. Testing Security Features:**

- App testing requires careful mocking of all dependencies (Router, Supabase, components)
- Focus on validation logic rather than implementation details
